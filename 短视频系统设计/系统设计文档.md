# 概要设计
## 实现原理
如何应对超高并发用户访问压力：
异地多IDC部署 + GSLB + CDN + 集中式缓存

如何存储海量的短视频文件：

每年新增视频量为：
```shell
550 x 3600 x 24 x 365 = 173亿
```
每年新增视频需要的存储空间为：
```shell
173亿 x 100M = 1700PB
```
1. 视频压缩, 虽然压缩不改变量级，但也能节约服务器成本
2. 多集群部署，比如每个idc部署100个HBase集群，每个集群只需要存储17PB的数据。 
3. 视频与集群的映射关系：根据hash(视频ID) % 100 的计算结果就是视频存储的集群ID。
